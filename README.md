## Суть и функционал:

Трекер расходов — консольное приложение для учёта ежедневных трат. Позволяет добавлять расходы, запрашивать сумму за период (с префиксными суммами — ответ за O(1)), искать день с максимальными тратами (линейный поиск), сортировать категории методом вставок, отменять последнее действие (через стек) и загружать данные из файла. Все данные хранятся в бинарном дереве поиска.

**Ключевые возможности:**

- Добавление расходов (день, сумма, категория)
- Сумма за период (день A — день B) — O(1)
- Отмена последнего действия — стек
- День с максимальными тратами — линейный поиск
- Сортировка категорий по сумме — вставками
- Загрузка из файла (100+ тестовых записей)
- Хранение данных — бинарное дерево

**Взаимодействие** с пользователем ведется через консольное меню: пользователь выбирает номер команды и дальше следует указаниям. Доступна загрузка данных из файла, в качестве примера приведен файл "expenses.txt" в папке "data". В этом файле все данные валидные для демонстрации функционала программы. Для тестового запуска рекомендуется сначала загрузить данные из файла при помощи команды "8". После загрузки можно добавлять еще траты.

**Внимание!** Программа работает с днями от 1 до 31. В месяцах с меньшим количеством дней дни 30-31 игнорируются.
	
## Структура программы:

### Программа состоит из 6 файлов (модулей), разделяющих ответственность между собой. 
	А именно:
	- main.py - точка входа пользовательский интерфейс.
	- models.py - классы данных (Expense, Category, DayExpense).
	- storage.py - класс Storage управляет деревом (связный список как дерево).
	- analytics.py - классы PrefixSumArray, MaxExpenseFinder, CategorySorter.
	- undo_stack.py - класс UndoStack (на основе стека).
	- utils.py - вспомогательные функции (валидация дат, форматирование)
	* Папка data содержит в себе демонстрационный файл с данными по тратам пользователя
	
### Использованные структуры данных и алгоритмы:

- Массив Префиксных сумм

- Линейный поиск максимума

- Сортировка вставками

- Стек

- Бинарное дерево поиска

## Логика кода

### Файл `models.py`

**Суть файла:**  
Определяет основные классы-сущности программы: отдельную трату (`Expense`), узел дерева (`DayExpense`) и категорию для отчётов (`Category`).

**Используемые структуры данных:**
- Пользовательские классы (`Expense`, `DayExpense`, `Category`)
- Внутри `DayExpense` — обычный список (`list`) для хранения трат одного дня
- Ссылки `left` / `right` для построения бинарного дерева поиска

---

### Файл `storage.py`

**Суть файла:**  
Реализует бинарное дерево поиска для хранения расходов по дням. Обеспечивает быструю вставку новой траты (O(log n)), поиск узла по дню и сбор всех расходов.

**Используемые структуры данных:**
- Бинарное дерево поиска (BST): каждый узел — объект `DayExpense`
- Ключ дерева — номер дня (`int` от 1 до 31)
- В каждом узле — список объектов `Expense`
- Рекурсивные методы для обхода, вставки и удаления

---

### Файл `analytics.py`

**Суть файла:**  
Содержит три класса для аналитической обработки данных:

1. `PrefixSumArray` — префиксный массив для ответа на запрос суммы за период за O(1)
2. `MaxExpenseFinder` — линейный поиск дня с максимальной суммой расходов
3. `CategorySorter` — сортировка категорий методом вставок

**Используемые структуры данных:**
- `PrefixSumArray`: обычный список фиксированной длины (32 элемента, индекс 0 не используется)
- `MaxExpenseFinder`: временные переменные (цикл и сравнения)
- `CategorySorter`: список объектов `Category` + классический алгоритм сортировки вставками

---

### Файл `undo_stack.py`

**Суть файла:**  
Реализует стек для отмены последнего добавленного расхода. Позволяет откатить операцию добавления за O(1).

**Используемые структуры данных:**
- Стек на основе **связного списка** (ручная реализация: класс `Node` с атрибутами `data` и `next`)
- Указатель на вершину стека (`top`)
- Счётчик размера (`size`)

---

### Файл `utils.py`

**Суть файла:**  
Набор вспомогательных функций для валидации данных, форматирования вывода, синхронизации между компонентами и загрузки из файла.

**Используемые структуры данных:**
- Обычные функции (без хранения состояния)
- Встроенные типы: `int`, `float`, `str`, `list`, `dict`
- Работа с файлами через `open()` и `pathlib`

---

### Файл `main.py`

**Суть файла:**  
Точка входа в программу. Реализует консольное меню, обрабатывает команды пользователя и координирует работу всех остальных модулей.

**Используемые структуры данных:**
- Главный цикл `while self.running`
- Условные операторы (`if/elif`) для выбора команды
- Объекты классов из других модулей (`ExpenseTree`, `PrefixSumArray`, `UndoStack` и др.)
- Словарь (`dict`) для хранения операции в стеке отмены
