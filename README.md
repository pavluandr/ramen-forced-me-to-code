## Структура программы:

Программа состоит из 6 файлов (модулей), разделяющих ответственность между собой. 
	А именно:
	- main.py - точка входа пользовательский интерфейс.
	- models.py - классы данных (Expense, Category, DayExpense).
	- storage.py - класс Storage управляет деревом (связный список как дерево).
	- analytics.py - классы PrefixSumArray, MaxExpenseFinder, CategorySorter.
	- undo_stack.py - класс UndoStack (на основе стека).
	- utils.py - вспомогательные функции (валидация дат, форматирование)
	
Использованные структуры данных и алгоритмы:
Массив Префиксных сумм
Линейный поиск максимума
Сортировка вставками
Стек
Бинарное дерево поиска

***Логика кода***
---
Файл models.py
		Суть файла:
			Определяет основные сущности (классы) программы - как выглядит отдельная трата, узел дерева и категория для отчетов.
		Используемые структуры данных:
			1. Пользовательские классы (Expense, DayExpense, Category)
			2. Внутри DayExpense - список или связный список для хранения трат одного дня
			3. Ссылки left/right для построения бинарного дерева
		Входные данные:
			Ничего не получает на вход как модуль. Классы получают параметры при создании объектов:
				Expense: день (int), сумма (float), категория (str).
				DayExpense: день (int), опционально первый расход (Expense).
				Category: название категории (str), общая сумма (float).
		Выходные данные:
			Объекты Expense, DayExpense, Category, которые используются в других модулях.

Файл analytics.py
		Суть файла:  
			Содержит три класса для аналитической обработки данных:
				1. Префиксный массив — быстрые ответы на запросы суммы за период (O(1)).
				2. Поиск дня с максимальным расходом — линейный алгоритм.
				3. Сортировка категорий — методом вставок.
		Используемые структуры данных:
				PrefixSumArray: обычный список Python (list) фиксированной длины (31 элемент).
				MaxExpenseFinder: временные переменные (цикл и сравнения).
				CategorySorter: список объектов Category + классический алгоритм сортировки вставками.
		Входные данные (что получает):
			PrefixSumArray.build(expenses_by_day) — массив сумм по дням (индекс = день, значение = сумма).
			MaxExpenseFinder.find_max_day(expenses_by_day) — массив сумм по дням.
			CategorySorter.build_category_totals(all_expenses) — список всех объектов Expense.
		Выходные данные (что отдаёт):
			 PrefixSumArray.range_sum(day_a, day_b) — число (float) — сумма расходов за период.
			 MaxExpenseFinder.find_max_day — кортеж (день, максимальная_сумма).
			 CategorySorter.insertion_sort() — отсортированный список объектов Category (по убыванию суммы).
			 CategorySorter.display_sorted_categories() — ничего (печатает на экран).

Файл undo_stack.py
		Суть файла:  
			Реализует стек для отмены последнего добавленного расхода. Позволяет откатить операцию добавления.
		Используемые структуры данных:
			Стек на основе связного списка (ручная реализация: узлы со ссылкой next).
		Входные данные:
			push(operation) — объект операции (например, словарь или кортеж с полями: тип операции и данные расхода).
		Выходные данные:
			pop() — последнюю добавленную операцию (или None, если стек пуст).
			is_empty() — булево значение (True/False).

Файл main.py
		Суть файла:
			Точка входа в программу. Реализует консольное меню, обрабатывает команды пользователя и координирует работу всех остальных модулей.
		Используемые структуры данных:
			Цикл while True.
		    Условные операторы (if`/`elif) для выбора команды.
		    Объекты классов из других модулей.
	    Входные данные (что получает):
		    Ввод пользователя из консоли (цифры команд, даты, суммы, категории).
	    Выходные данные (что отдаёт):
		    Приглашения и подсказки в консоль.
		    Результаты запросов (суммы за период, максимальный расход, отсортированные категории).
		    Сообщения об ошибках (неверный день, отрицательная сумма и т.д.).
		    
Файл storage.py
	Суть файла:
		Реализует бинарное дерево поиска для хранения расходов по дням. Обеспечивает быструю вставку новой траты и поиск трат за конкретный день.
	Используемые структуры данных:
		Бинарное дерево поиска (BST): каждый узел — объект DayExpense. 
		Ключ дерева — номер дня (int).
		 В каждом узле — список (или связный список) объектов Expense.
    Входные данные:
	    insert(day, expense) — номер дня и объект Expense
	    find(day) — номер дня для поиска
	    get_all_expenses() — ничего, только запрос
    Выходные данные:
	    insert — ничего (изменяет дерево)
	    find — узел `DayExpense` или `None`, если день не найден
	    get_all_expenses — список всех объектов `Expense` из всех дней (например, для аналитики)
    
Файл utils.py
	Суть файла:
		Набор вспомогательных функций для валидации, форматирования вывода и синхронизации между компонентами.
	Используемые структуры данных:
		Обычные функции (без хранения состояния).
    Входные данные (что получает):
	    validate_day(day) — число (int).
	    validate_amount(amount) — число (float или int).
	    format_currency(value) — число.
	     get_date_range_input() — ничего (читает из консоли).
	     sync_prefix_sum_with_tree(tree, prefix_sum_obj) — объект дерева и объект префиксного массива.
    Выходные данные (что отдаёт):
	    validate_day — True/False.
	    validate_amount — True/False.
	     format_currency — строка с отформатированной суммой (например, "250.50 руб.").
	    get_date_range_input — кортеж (day_a, day_b).
	    sync_prefix_sum_with_tree — ничего (обновляет состояние префиксного массива).
